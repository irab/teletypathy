# Teletypathy Diagrams

This directory contains diagrams explaining how the Teletypathy system works, both from technical and user perspectives.

## Diagram Files

### Technical Diagrams

- **`technical_system_architecture.md`** - Complete technical system architecture
  - System architecture overview
  - Hardware block diagrams
  - Protocol stack
  - Encoding system architecture

- **`data_flow.md`** - Data flow through the system
  - End-to-end data flow
  - Message flow for text
  - Pattern execution timeline

### User-Facing Diagrams

- **`user_experience.md`** - User experience and interaction
  - How Teletypathy works (simple explanation)
  - User journey
  - What patterns feel like
  - Learning progression
  - Use cases
  - Device placement

- **`comprehension_rates.md`** - Performance comparison
  - Comparison to speech and reading
  - Words per minute (WPM) and characters per second (CPS) rates
  - Learning curve comparison
  - Realistic performance expectations
  - Use case scenarios

## Diagram Formats

All diagrams are created using ASCII art for maximum compatibility. They can be:
- Viewed directly in markdown viewers
- Converted to images using tools like `ascii-image-converter`
- Used as reference for creating visual diagrams

## Usage

These diagrams are intended for:
- **Documentation**: Explaining the system to developers and users
- **Presentations**: Visual aids for explaining the concept
- **Onboarding**: Helping new contributors understand the system
- **User guides**: Helping users understand how to use the system
